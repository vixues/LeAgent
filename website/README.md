# LeAgent Website

Static marketing site built with React 19 + Vite + TypeScript + Tailwind CSS 4.

## Development

```bash
cd website
npm install
npm run dev        # http://localhost:8080
npm run build      # production build -> dist/
npm run preview    # preview production build
npm run typecheck  # type-check without emitting
npm run check:links  # verify external URLs in source
```

`npm run build` / `npm run dev` first run `prepare:static` + `seo`, which:

- Copy `../scripts/install.{sh,ps1,bat}` → `public/` (site root installers)
- Copy `../docs/assets/**` → `public/docs/assets/**` (README logo / hero — required for offline `dist` deploy)
- Generate `robots.txt`, `sitemap.xml`, `agents.txt`, `agents.json`, `llms.txt`

Tutorial **markdown bodies** (面试 / Agent / 技术资料 / README) are **bundled into the JS** at build time via `import.meta.glob`. Uploading only `dist/` is enough — you do **not** need the git repo next to the site.

## Routing & SEO

- The site uses **BrowserRouter** (path URLs like `/tutorials/agent/01-agent-vs-chatbot`), not hash routes.
- Legacy `/#/about`-style links are redirected to `/about` via a boot script in `index.html`.
- `npm run seo` (also run by `prebuild`) writes `public/robots.txt`, `sitemap.xml`, `agents.txt`, `agents.json`, and `llms.txt`, including all tutorial article URLs scanned from `../docs/`.
- Tutorial markdown is imported at build time from the repo (`docs/interview`, `docs/guides/agent`, `docs/technical`, `README*.md`).

## Production deployment (upload `dist/` only)

1. Build on a machine that has the full repo (so docs can be imported):

```bash
cd website
npm install
npm run build
```

2. Upload the **entire** `dist/` directory to your web server root (Nginx, 宝塔, etc.). That folder is self-contained: HTML/CSS/JS + images + install scripts + SEO files + README assets under `docs/assets/`.

3. Configure Nginx so install scripts are served as plain files **before** the SPA fallback. Otherwise `try_files` may return `index.html` for `/install.sh`, and `curl | bash` will fail:

```nginx
# Install scripts: serve files directly (no SPA fallback)
location ~ ^/install\.(sh|ps1|bat)$ {
    root /path/to/dist;
    default_type text/plain;
    add_header Cache-Control "public, max-age=300";
}

location / {
    root /path/to/dist;
    try_files $uri $uri/ /index.html;
}
```

4. Smoke-check after deploy:

```bash
curl -fsSI https://your.domain/install.sh | grep -E 'HTTP|content-type'
curl -fsSI https://your.domain/docs/assets/logo.svg | head -5
# Open in browser: /  /tutorials  /tutorials/intro
```

Note: decorative README badges (shields.io / GitHub Actions) still load from the public internet; core pages, tutorial text, logo, and hero image do not need GitHub.

```bash
curl -fsSL https://vixues.com.cn/install.sh | bash
```

## Customization

- **Contact details**: edit `src/lib/content.ts` -> `CONTACT` object.
- **Site / repo URLs**: copy `.env.example` to `.env` and set `VITE_SITE_ORIGIN`, `VITE_REPO_URL`.
- **Favicon**: replace `public/favicon.svg`.
- **Images**: drop screenshots into `public/images/`.

## Tutorials / docs

Marketing routes use **BrowserRouter** (path URLs like `/tutorials/...`).
Nginx must keep the SPA `try_files` fallback so deep links refresh correctly.

Tutorial markdown is loaded at build time from the repo (`docs/interview`,
`docs/guides/agent`, `docs/technical`, `README*.md`) via `import.meta.glob`.
SEO files (`sitemap.xml`, `llms.txt`, `agents.txt`) are regenerated by
`npm run seo` and include all tutorial article URLs.

Legacy hash URLs (`/#/about`) are redirected to path URLs in `index.html`.

## i18n

- Bilingual: zh-CN (default) and en-US.
- All copy lives in `src/i18n/translations.ts`.
- Language toggle persists to `localStorage` key `leagent-site-language`.
- Interview / Agent guides are Chinese-only; en-US shows a notice and zh body.

## Themes

- Dark (default) and Light themes.
- Toggle persists to `localStorage` key `leagent-site-theme`.
- CSS tokens in `src/styles/index.css` under `html[data-theme="dark"|"light"]`.
- Pre-hydration script in `index.html` prevents flash on reload.

## External link check

`npm run check:links` scans `src/` and `index.html` for `https://` URLs and probes each with HEAD/GET. GitHub links may fail until the repository is public; placeholder contact URLs in `content.ts` will fail until updated.
