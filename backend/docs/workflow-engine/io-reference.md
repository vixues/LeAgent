# IO reference — canonical workflow document

Every workflow stored in the database, generated by templates, or submitted
through the import endpoint must use the **canonical shape**. The shape is
enforced at load time by `leagent.workflow.io.loader._assert_canonical`
and there is no runtime migration layer.

## Top-level structure

```jsonc
{
  "id": "my_flow",
  "name": "My Flow",
  "description": "...",
  "inputs": [],            // declared flow inputs (typed primitives)
  "outputs": [],           // declared flow outputs
  "metadata": {},          // free-form tags, category, etc.

  "nodes": {               // map of node_id -> node triple
    "<node_id>": {
      "class_type": "StartNode",   // must resolve in the NodeRegistry
      "inputs":  { /* widget/input values keyed by input id */ },
      "meta":    { /* optional node-local metadata (name, description) */ },
      "control": { /* control-flow directives, see below */ }
    }
  },

  "control": {
    "start":        "<node_id>",          // entry node
    "end":          "<node_id>",          // terminal node
    "edges":        [ /* explicit data edges */ ],
    "timeout_sec":  3600,
    "max_retries":  3,
    "tags":         []
  }
}
```

Key rules:

- `nodes` is a **mapping** (dict), not a list. Every node entry has a
  `class_type` that must be registered.
- `control` is always a mapping. Missing keys get sane defaults.
- `schema_version` / `version` fields are forbidden — the loader rejects
  documents that include them.

## Node `control` directives

```jsonc
{
  "next":          "<node_id>",        // next node after success
  "error_handler": "<node_id>",        // on exception
  "else_node":     "<node_id>",        // Condition: false branch
  "on_reject":     "<node_id>",        // HumanReview rejection
  "conditions": [                      // Condition: ordered branches
    { "when": { "left": "...", "op": "eq", "right": "..." },
      "then_node": "<node_id>" }
  ]
}
```

## Authoring surfaces

The following formats are accepted **at write time** (create/update, YAML
template load, starter-project bootstrap) and are converted to the
canonical shape via `leagent.workflow.io.authoring.to_canonical`:

- Flat lists: `{ "nodes": [{ "id": "...", "type": "tool_call", "next": "..." }, ...] }`.
- YAML templates using mustache placeholders (`{{expr}}`) — placeholders are
  rewritten to `${expr}` during canonicalization.
- Visual-editor graphs (`FlowData`) emitted by the React flow canvas.

Read-time the engine only understands the canonical shape, which keeps the
scheduler simple and the cache key deterministic.

## `Schema` (node metadata)

Each node class exposes a `Schema` dataclass via `define_schema()`:

```python
Schema(
    node_id="MyNode",
    display_name="My Node",
    category="custom",
    inputs=[IO.String.Input(id="text", default="")],
    outputs=[IO.String.Output(id="echo")],
    # Contract flags:
    is_output_node=False,      # terminal UI output
    not_idempotent=False,      # bypass caches when True
    enable_expand=False,       # allow NodeOutput.expand
    is_input_list=False,       # accept list-typed inputs
    accept_all_inputs=False,   # catch-all socket
    has_intermediate_output=False,
    input_order=[],            # UI ordering
    output_tooltips=[],        # UI hint per output
)
```

## Input primitives

`IO.*` produces typed `WidgetInput` subclasses with rich UI/validation
options. See `leagent/workflow/io/types.py` for the full set; highlights:

- `IO.String.Input(default, multiline, placeholder, dynamic_prompts, pattern, min_length, max_length)`
- `IO.Int.Input(default, min, max, step, display)`
- `IO.Float.Input(default, min, max, step, display, round)`
- `IO.Boolean.Input(default, label_on, label_off)`
- `IO.Combo.Input(choices, multiselect, remote)`
- `IO.File.Input(accept)`

All primitives implement `validate(value)` for runtime checks.

## Contract hooks

Two hooks on `WorkflowNode` drive caching + lazy evaluation:

- `fingerprint_inputs(**resolved_inputs)` — return a deterministic string
  used as a cache key fragment, or `NOT_CACHEABLE` to bypass caches.
- `check_lazy_status(**resolved_inputs)` — return a list of input ids that
  still need resolution before `execute()` can run; used by the runner to
  implement lazy-input semantics.

Both have sensible defaults in `leagent.workflow.io.contract`.
